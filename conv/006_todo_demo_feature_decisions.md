# Todo 데모 기능 보강

## 배경
- `feat-mini-react-impl` 폴더 안 Todo 앱을 데모용으로 보여줄 때, 입력 초기화 문제와 수정 기능 부재 때문에 동작 설명이 매끄럽지 않은 상태였다.
- mini-react의 hook 동작을 설명하려면 내부 상태를 눈으로 확인할 수 있는 화면이 필요했다.

## 요청 내용
- Todo 추가 후 입력창 초기화
- 생성된 Todo 텍스트 수정 기능
- 필터링 기능 추가
- hooks 배열을 화면에서 볼 수 있는 디버그 패널 추가

## 고려한 선택지
- 각 TodoItem 안에 개별 편집 상태를 두는 방법
- 루트 `App`에서 편집 상태와 필터 상태를 모두 관리하는 방법
- hooks 정보를 콘솔에만 출력하는 방법
- hooks 정보를 화면 패널로 보여주는 방법

## 결정
- 현재 mini-react 구조에서는 함수 컴포넌트별 독립 인스턴스보다 루트 렌더 단위 관리가 더 안전하다고 보고, 편집 상태와 필터 상태를 `App`에 모아서 처리했다.
- 입력 초기화가 실제 화면에 반영되도록 DOM attribute뿐 아니라 `input.value`, `input.checked` 같은 DOM property도 함께 갱신하도록 patch 로직을 보강했다.
- 데모 설명 편의를 위해 hooks 배열은 콘솔이 아니라 화면 하단 패널에서 즉시 볼 수 있게 했다.

## 영향
- Todo 추가, 수정, 필터링 흐름을 한 화면에서 바로 시연할 수 있게 됐다.
- hook slot 상태를 UI에서 바로 보여줄 수 있어, `useState`, `useMemo`, `useEffect`가 어떤 순서로 저장되는지 설명하기 쉬워졌다.
