# PRD 구조 재편 및 conv 작성 방식 정립

## 배경

첫 번째 PRD 작성 시 conv/ 파일 5개를 기술 명세(타입 정의, 알고리즘 설명, 코드 예시) 중심으로 작성했다.
그런데 이는 docs/에 들어가야 할 내용이었다.

## 요청 내용

"conv 문서를 합쳐서 docs에 한 파일로 합친 PRD 문서를 생성하고,
conv 파일에는 내가 너한테 명령한 배경이나 그 의사결정의 이유 등이 들어있어야 해.
따라서 기존의 conv 파일은 제거하고(하기전에 커밋)
conv을 앞으로 잘 작성할 수 있도록 Claude.md 파일을 생성하고 해당 설명을 추가."

## 결정

**역할 분리:**

| 폴더/파일 | 내용 |
|---|---|
| `docs/` | 기술 명세, PRD, 알고리즘 설명 (What / How) |
| `conv/` | 의사결정 배경, 요청 맥락, 선택 이유 (Why) |
| `CLAUDE.md` | Claude Code 행동 규칙 |

기존 conv/ 5개 파일의 기술 명세는 `docs/002_PRD.md`로 통합.
CLAUDE.md에 conv/ 작성 규칙을 명문화해서 앞으로 올바른 형식을 유지.

## 영향

이후 모든 conv/ 파일은 "왜"에 집중한다.
기술 스펙을 conv/에 쓰면 중복 문서가 생기고 유지보수가 어려워진다.
