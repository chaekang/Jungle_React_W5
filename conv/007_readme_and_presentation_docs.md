# README 초안 및 발표 대본 문서화 결정

## 배경

Week 5 과제는 구현 결과물뿐 아니라 구현 원리 설명, 테스트 검증, 그리고 목요일 오전 발표까지 포함한다.
발표 자료는 별도 슬라이드가 아니라 저장소의 README.md를 기준으로 설명해야 하므로, 코드만 있는 상태로는 전달력이 부족할 수 있었다.

## 요청 내용

과제 안내문을 바탕으로 프로젝트 README와 발표 대본을 작성하고, 우선 텍스트 파일 형태로 정리해 달라는 요청이 있었다.

## 고려한 선택지

1. 바로 `README.md`를 완성본으로 작성한다.
2. 먼저 텍스트 초안을 작성한 뒤, 추후 README.md와 발표 자료용으로 다듬는다.
3. 발표 대본 없이 README만 작성한다.

## 결정

우선 `docs/`에 텍스트 초안 두 개를 작성하기로 했다.

- `006_readme_draft.txt`: README 구조와 핵심 메시지를 먼저 정리
- `007_presentation_script.txt`: 4분 발표 흐름과 QnA 대비 문장 정리

즉시 `README.md`로 확정하지 않은 이유는, 이후 팀 합의에 따라 팀명, 실행 캡처, 역할 분담, 최종 데모 순서가 달라질 수 있기 때문이다.
텍스트 초안으로 먼저 만들면 빠르게 검토하고 수정한 뒤 최종 README.md로 옮기기 쉽다.

## 영향

- 발표 준비를 README 중심으로 빠르게 진행할 수 있다.
- 구현 설명, 차별점, 테스트 전략을 한 번에 정리해 팀 내부 공통 설명 자료로 쓸 수 있다.
- 이후 README.md 최종본으로 변환할 때 내용 손실 없이 반영할 수 있다.
