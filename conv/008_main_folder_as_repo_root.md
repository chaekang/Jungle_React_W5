# main 폴더를 저장소 루트로 맞추는 결정

## 배경

현재 로컬 작업은 `Jungle_React_W5/main` 폴더 안에서 진행되고 있었지만, 상위 폴더 `Jungle_React_W5`가 Git 저장소 루트였다.
이 상태로 푸시하면 GitHub 저장소 루트에 `main/` 폴더가 생기고, 현재 작업 파일들은 그 안에 들어가게 된다.

## 요청 내용

현재 작업 중인 `main` 폴더 안의 파일들만 그대로 GitHub `main` 브랜치 루트에 올리고 싶다는 요청이 있었다.

## 고려한 선택지

1. 상위 저장소 구조를 유지한 채 `main/` 폴더째 푸시한다.
2. 상위 저장소에서 파일을 루트로 이동한 뒤 푸시한다.
3. 현재 `main` 폴더를 독립 Git 저장소로 만들고, 원격 `main` 브랜치와 직접 연결한다.

## 결정

현재 `main` 폴더를 독립 Git 저장소로 초기화하고, 기존 GitHub 원격 저장소의 `main` 브랜치 히스토리를 이어받는 방식으로 연결하기로 했다.

이 방식을 택한 이유는 다음과 같다.

- 현재 작업 폴더 구조를 그대로 유지할 수 있다.
- 상위 저장소 파일을 이동하거나 정리하는 파괴적 작업이 필요 없다.
- 원격 `main` 브랜치와 연결해 `main` 폴더 내부 파일이 저장소 루트 기준으로 보이게 만들 수 있다.

## 영향

- 이후 `main` 폴더 안에서 Git 명령을 실행하면 이 폴더가 독립 프로젝트처럼 동작한다.
- GitHub 저장소에서는 `README.md`, `src`, `docs`, `tests` 등이 루트에 직접 보이게 된다.
- 상위 폴더 저장소와는 별개의 Git 기준점을 갖게 되므로, 앞으로는 어느 저장소 기준으로 작업하는지 구분이 필요하다.
