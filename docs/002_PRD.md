# PRD: Mini-React (Week 5)

## 개요

React의 핵심 개념(Component, State, Hooks, Virtual DOM)을 외부 프레임워크 없이 TypeScript로 처음부터 구현하고, 이를 활용한 Todo List 데모 앱을 제작한다.

- **기술 스택:** TypeScript + Vite + Vitest
- **데모 앱:** Todo List
- **발표:** 목요일 오전 (4분 데모 + 1분 Q&A)

---

## 빌드 설정

### JSX Pragma

JSX 문법(`<div>`, `<button>` 등)을 우리의 `h()` 함수로 변환하도록 설정한다.

**`tsconfig.json`**
```json
{
  "compilerOptions": {
    "jsx": "react",
    "jsxFactory": "h",
    "jsxFragmentFactory": "Fragment"
  }
}
```

**`vite.config.ts`**
```ts
export default defineConfig({
  esbuild: { jsxFactory: 'h', jsxFragment: 'Fragment' }
});
```

각 컴포넌트 파일 상단에 `import { h } from '../mini-react'`를 반드시 포함해야 한다.

## 모듈 명세

### 1. VNode (`src/mini-react/vdom.ts`)

```ts
export interface VElement {
  kind: 'element';
  type: string;                   // 'div', 'button', etc.
  props: Record<string, unknown>;
  children: VNode[];
  key?: string | number;
}

export interface VText {
  kind: 'text';
  text: string;
}

export type VNode = VElement | VText;
```

**`createElement` / `h()` 동작:**
- 문자열/숫자 자식 → `VText` 자동 변환
- `null` / `undefined` 자식 → 제거
- 중첩 배열 → 1단계 flatten
- `key` prop → `VElement.key`로 분리 (props에서 제거)

---

### 2. Diff (`src/mini-react/diff.ts`)

**PatchOp 타입:**
```ts
export type PatchOp =
  | { type: 'REPLACE';       node: VNode }
  | { type: 'UPDATE_TEXT';   text: string }
  | { type: 'UPDATE_PROPS';  added: Record<string, unknown>; removed: string[] }
  | { type: 'APPEND';        node: VNode }
  | { type: 'REMOVE' }
  | { type: 'CHILDREN';      childPatches: ChildPatch[] }

export interface ChildPatch { index: number; op: PatchOp; }
```

**결정 트리:**
1. `old.kind !== new.kind` → `REPLACE`
2. 둘 다 element이고 `old.type !== new.type` → `REPLACE`
3. 둘 다 VText → 텍스트가 다를 때만 `UPDATE_TEXT`
4. 둘 다 동일 태그 VElement:
   - props 변경분 → `UPDATE_PROPS`
   - children zip (인덱스 기반) → `CHILDREN`
   - 자식 추가분 → `APPEND`, 삭제분 → `REMOVE`

---

### 3. Patch (`src/mini-react/patch.ts`)

`PatchOp[]`를 받아 실제 DOM에 최소한의 변경만 적용한다.

**이벤트 핸들러 관리:**

```ts
const listenerMap = new WeakMap<Node, Map<string, EventListener>>();
```

- prop 이름 `/^on[A-Z]/` 패턴으로 이벤트 핸들러 감지
- `'onClick'` → `addEventListener('click', ...)` 변환
- 재렌더 시 이전 리스너 제거 후 새 리스너 등록 (중복 방지)
- `WeakMap` 사용으로 DOM 노드 GC 시 자동 해제

**`createDom(vnode)`:** VNode에서 실제 DOM 노드 생성. `mount()`와 `REPLACE`/`APPEND` op에서 사용.

---

### 4. Hooks (`src/mini-react/hooks.ts`)

#### Global Cursor Pattern

```ts
let currentComponent: FunctionComponent | null = null;
let hookIndex: number = 0;
```

렌더 함수 실행 전 `setCurrentComponent(comp)`로 포인터를 설정하고, 각 훅은 `hookIndex`로 `comp.hooks[]` 슬롯을 찾는다.

#### useState

슬롯: `{ type: 'state', value: T }`

```ts
useState<T>(initialValue: T): [T, (next: T | ((prev: T) => T)) => void]
```

- 첫 렌더: `initialValue`로 슬롯 초기화
- `setState(next)`: 슬롯 값 업데이트 후 `comp.update()` 호출
- 함수형 업데이터(`setState(prev => ...)`) 지원

#### useEffect

슬롯: `{ type: 'effect', deps, cleanup, pendingFn }`

```ts
useEffect(fn: () => (() => void) | void, deps: unknown[]): void
```

- deps 얕은 비교 (`Object.is` element-by-element)
- DOM patch 완료 후 microtask로 실행: `Promise.resolve().then(flushEffects)`
- cleanup은 다음 effect 실행 직전에 호출

#### useMemo

슬롯: `{ type: 'memo', deps, value: T }`

```ts
useMemo<T>(fn: () => T, deps: unknown[]): T
```

- deps 변경 시에만 `fn()` 재실행, 동기 반환

---

### 5. FunctionComponent (`src/mini-react/component.ts`)

```ts
class FunctionComponent {
  hooks: HookSlot[] = [];
  private vdom: VNode | null = null;
  private container: HTMLElement;
  private renderFn: () => VNode;

  mount(): void   // 첫 렌더: createDom으로 직접 DOM 생성
  update(): void  // 재렌더: diff → patch
}
```

**구현 주의점:**
- `setCurrentComponent` / `clearCurrentComponent`는 반드시 `try/finally`로 감쌀 것
- `mount()`는 diff 없이 직접 `createDom()` 호출
- `flushEffects()`는 `patch()` 완료 후 microtask로 지연 실행

## Todo 앱 설계 (`src/app/`)

### 도메인 타입

```ts
// types.ts
export interface Todo {
  id: number;
  text: string;
  completed: boolean;
}
```

### 컴포넌트 구성 (Lifting State Up)

```
App (useState, useEffect, useMemo)
 ├── TodoInput    ← value, onInput, onAdd
 ├── TodoList     ← todos, onToggle, onDelete
 │   └── TodoItem (×n)
 └── TodoFooter   ← remaining, total
```

### App 상태 및 훅

```ts
const [todos, setTodos]       = useState<Todo[]>([]);
const [inputText, setInputText] = useState('');

const remainingCount = useMemo(
  () => todos.filter(t => !t.completed).length,
  [todos]
);

useEffect(() => {
  document.title = `Todos (${remainingCount} left)`;
}, [remainingCount]);
```

### Props 명세

| 컴포넌트 | Props |
|---|---|
| `TodoInput` | `value`, `onInput(v: string)`, `onAdd()` |
| `TodoList` | `todos`, `onToggle(id)`, `onDelete(id)` |
| `TodoItem` | `todo`, `onToggle(id)`, `onDelete(id)` |
| `TodoFooter` | `remaining`, `total` |
