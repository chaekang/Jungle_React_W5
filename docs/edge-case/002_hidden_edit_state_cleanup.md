# 엣지케이스 수정 기록

## 1. 제목
- 편집 중인 Todo가 화면에서 사라질 때 숨겨진 편집 상태를 정리한 수정
- 예: 빈 입력 처리 누락으로 인한 예외 수정

## 2. 배경
- 문제가 발생한 기능: Todo 편집 상태 관리, 필터 전환, 완료 토글, 편집 중 삭제
- 문제를 발견한 상황: 수정 중인 Todo가 필터 변경이나 완료 상태 변경으로 현재 목록에서 사라지거나, 편집 도중 바로 삭제될 때
- 일반 케이스에서는 왜 드러나지 않았는지: 보통은 같은 필터 화면 안에서 수정 후 바로 저장/취소를 끝내기 때문에, 편집 상태가 보이지 않게 되는 조합이 자주 발생하지 않았다

## 3. 재현 조건
- 입력값: 수정 중인 임시 draft 문자열
- 사용자 행동: 수정 진입 후 필터 변경, 체크박스로 완료 상태 토글, 편집 중 `삭제` 버튼 클릭
- 실행 환경: `TypeScript + Vite + Vitest`, 브라우저 렌더 환경과 jsdom 테스트 환경
- 재현 절차:
  1. Todo 항목에서 `수정` 버튼을 눌러 편집 모드에 진입한다.
  2. draft를 입력한 뒤 현재 목록에서 해당 Todo가 보이지 않도록 필터를 바꾸거나 완료 상태를 바꾸거나 삭제한다.
  3. 목록이 바뀐 뒤 편집 입력창과 편집 상태가 남아 있는지 확인한다.

## 4. 기대 동작
- 편집 중인 Todo가 현재 화면에서 사라지면 숨겨진 편집 상태가 남지 않아야 한다.
- 편집 중인 Todo를 삭제하면 입력창과 편집 상태가 함께 정리되어야 한다.
- 다시 필터를 원래대로 돌려도 저장되지 않은 draft가 남아 있지 않아야 한다.

## 5. 실제 동작
- 편집 대상이 필터 변경이나 완료 상태 변경으로 현재 목록에서 사라져도 `editingId`, `editingText`가 남아 있을 수 있었다.
- 사용자는 화면에 입력창이 없는데 내부적으로는 편집 상태가 살아 있는 상황을 만들 수 있었다.
- 편집 중 삭제는 일부 상태만 정리되기 때문에, 숨겨진 편집 상태와 결합되면 추적이 어려웠다.

## 6. 원인 분석
- 직접 원인: 현재 편집 대상이 `todos`와 `filter` 기준으로 여전히 화면에 존재하는지 점검하는 후속 정리 로직이 없었다.
- 근본 원인: 루트 `App`이 모든 편집 상태를 들고 있으면서도, 편집 상태의 종료 조건을 저장/취소에만 연결해 두었다.
- 기존 로직이 놓친 이유: 편집 상태의 생명주기를 사용자 액션 한두 개로만 생각했고, 필터/토글/삭제 같은 다른 상태 변화와의 조합을 별도 시나리오로 보지 않았다.

## 7. 수정 내용
- 수정한 파일:
  - `src/app/App.tsx`
  - `tests/app.test.ts`
- 수정한 함수/모듈:
  - `App()` 내부 편집 상태 정리 effect
  - 편집 상태 소멸 시나리오 테스트
- 변경 내용:
  - `App.tsx`에 현재 `editingId`가 가리키는 Todo가 삭제되었는지, 현재 `filter`에서 보이는지 검사하는 effect를 추가했다.
  - 편집 대상이 삭제되었거나 현재 필터에서 보이지 않으면 `editingId`와 `editingText`를 초기화하도록 했다.
  - 상태 정리가 일어나는 경우 이벤트 로그도 남기도록 했다.
  - `tests/app.test.ts`에 필터 변경, 완료 토글, 편집 중 삭제 시나리오를 대량으로 추가했다.

## 8. 영향 범위
- 영향 받는 기능: Todo 편집 상태, 필터 전환, 완료 토글, 편집 중 삭제
- 영향 없다고 판단한 범위: 정상 저장 성공 경로, Todo 추가 기능, mini-react 코어의 diff/patch 기본 동작
- 주의할 점: 이번 수정은 “숨겨진 편집 상태 정리”에 초점을 둔 것이며, 여러 `setState`에 따른 배칭 문제는 그대로 남아 있다.

## 9. 검증 내용
- 테스트한 케이스:
  - 정상 케이스:
    - 필터가 걸린 상태에서도 정상 편집 저장
    - 편집 후 저장/취소가 정상 종료되는지 확인
  - 엣지케이스:
    - 편집 중인 활성 Todo가 `completed` 필터에서 사라지는 경우
    - 편집 중인 완료 Todo가 `active` 필터에서 사라지는 경우
    - 편집 중인 Todo가 체크박스 토글로 현재 필터에서 사라지는 경우
    - 편집 중인 Todo를 바로 삭제하는 경우
  - 회귀 테스트:
    - 필터를 다시 `전체`로 돌렸을 때 이전 draft가 남지 않는지 확인
    - 삭제 후 해당 Todo가 실제 목록에서 제거되었는지 확인
- 테스트 결과:
  - 편집 상태 정리 관련 앱 시나리오 60개 통과
  - 전체 테스트 145개 통과
  - `npm run build` 통과

## 10. 남은 이슈 / 후속 작업
- 편집 대상이 사라질 때 사용자에게 “편집이 자동 종료되었다”는 UI 메시지를 별도로 보여주지는 않는다.
- 향후 다중 선택 편집이나 임시 draft 복구 기능이 필요하면 현재 정리 정책을 다시 설계해야 한다.
