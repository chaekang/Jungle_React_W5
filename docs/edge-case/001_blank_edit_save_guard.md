# 엣지케이스 수정 기록

## 1. 제목
- 공백만 있는 수정값 저장 시도를 UI 단계에서 차단한 수정
- 예: 빈 입력 처리 누락으로 인한 예외 수정

## 2. 배경
- 문제가 발생한 기능: Todo 편집 저장 기능
- 문제를 발견한 상황: 수정 모드에서 입력값을 전부 지운 뒤 `저장` 버튼이나 Enter로 저장하려고 할 때
- 일반 케이스에서는 왜 드러나지 않았는지: 보통은 유효한 문자열로 수정한 뒤 저장하기 때문에, 공백만 남긴 상태에서 저장을 시도하는 조합이 자주 나오지 않았다

## 3. 재현 조건
- 입력값: `''`, `' '`, `'   '`, `'\t'`, `'\n'`, 공백 조합 문자열
- 사용자 행동: 수정 진입 후 입력값을 공백만 남기고 `저장` 버튼 클릭 또는 Enter 입력
- 실행 환경: `TypeScript + Vite + Vitest`, 브라우저 렌더 환경과 jsdom 테스트 환경
- 재현 절차:
  1. Todo 항목에서 `수정` 버튼을 눌러 편집 모드에 진입한다.
  2. 입력값을 공백만 남기도록 수정한다.
  3. `저장` 버튼을 누르거나 Enter를 입력한다.

## 4. 기대 동작
- 공백만 있는 수정값은 저장되지 않아야 한다.
- 사용자는 현재 저장이 불가능한 상태라는 것을 즉시 알 수 있어야 한다.
- 기존 Todo 텍스트는 유지되어야 한다.

## 5. 실제 동작
- 저장 로직 내부에서는 공백 저장을 막고 있었지만, 버튼은 그대로 활성화되어 있었다.
- 사용자는 `저장`을 눌러도 아무 반응이 없는 것처럼 느낄 수 있었다.
- Enter 입력도 겉보기에는 동작 시도처럼 보이지만 실제 반영은 되지 않았다.

## 6. 원인 분석
- 직접 원인: 공백 저장 방어가 `handleEditSave()` 내부에만 있었고, UI 계층에서는 저장 가능 여부를 표시하지 않았다.
- 근본 원인: 저장 유효성 검사를 “상태 전이 가드”로만 두고, “사용자 피드백이 있는 UI 상태”로 연결하지 않았다.
- 기존 로직이 놓친 이유: 정상 저장 성공 경로 위주로 검증했고, 저장 불가 상태에서의 버튼/Enter 동작을 따로 테스트하지 않았다.

## 7. 수정 내용
- 수정한 파일:
  - `src/app/components/TodoItem.tsx`
  - `src/app/styles.css`
  - `tests/app.test.ts`
- 수정한 함수/모듈:
  - `TodoItem`
  - `.todo-action:disabled` 스타일
  - 앱 편집 저장 시나리오 테스트
- 변경 내용:
  - `TodoItem.tsx`에서 `editingText.trim().length === 0`일 때 `저장` 버튼을 비활성화했다.
  - 같은 조건에서 Enter 저장도 실행되지 않도록 막았다.
  - `styles.css`에 비활성화된 액션 버튼의 opacity/cursor 스타일을 추가했다.
  - `tests/app.test.ts`에 공백 조합별 버튼 저장/Enter 저장 차단 케이스를 추가했다.

## 8. 영향 범위
- 영향 받는 기능: Todo 편집 저장 버튼 상태, Enter 저장 처리, 편집 입력 유효성
- 영향 없다고 판단한 범위: 정상적인 Todo 추가/삭제/토글, mini-react 코어 로직
- 주의할 점: 현재는 저장 불가 상태를 버튼 비활성화로만 표현하며, 별도의 에러 메시지나 안내 문구는 없다.

## 9. 검증 내용
- 테스트한 케이스:
  - 정상 케이스:
    - 유효한 문자열을 `저장` 버튼으로 저장
    - 유효한 문자열을 Enter로 저장
  - 엣지케이스:
    - 빈 문자열 저장 시도
    - 공백 문자열 저장 시도
    - 탭/개행 포함 공백 문자열 저장 시도
    - 공백 상태에서 `저장` 버튼 클릭
    - 공백 상태에서 Enter 입력
  - 회귀 테스트:
    - 저장 후 편집 모드가 정상 종료되는지 확인
    - 저장 후 버튼 구조가 정상 복원되는지 확인
- 테스트 결과:
  - 공백 저장 방지 관련 앱 시나리오 30개 통과
  - 전체 테스트 145개 통과
  - `npm run build` 통과

## 10. 남은 이슈 / 후속 작업
- 저장 불가 이유를 사용자에게 더 직접적으로 알려주는 안내 문구는 아직 없다.
- 입력 길이 제한, 특수문자 처리, 중복 텍스트 정책은 아직 정의하지 않았다.
