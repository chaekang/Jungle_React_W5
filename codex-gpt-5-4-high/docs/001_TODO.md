# 수요 코딩회 (수요일)

## 목적

- 단 하루, AI를 도구로 삼아 팀별 프로젝트에 몰입합니다.
- 결과물만큼 과정도 중요합니다. AI가 작성한 코드를 자신의 것으로 이해하고, 원리를 설명할 수 있어야 합니다.
- 낯선 기능도 Top-down으로 빠르게 풀어내며 학습 민첩성을 높이고, 현업 수준의 기술을 직접 다루며 실무 감각을 익힙니다.

## 요구사항

- 이번 주 과제는 `React`의 핵심 개념인 `Component`, `State`, `Hooks`를 직접 구현하고, 이를 활용해 동작하는 웹 페이지를 만드는 것입니다.

### 1. Component

- `Component`는 UI를 나누는 작은 단위입니다.
- 각 `Component`는 자신의 로직과 렌더링을 담당합니다.
- 반드시 `함수형 컴포넌트`로 구현합니다.

#### 구현 방식

- 함수형 컴포넌트를 감싸는 `FunctionComponent` 클래스를 직접 구현합니다.
- 이 클래스에는 다음 기능이 필요합니다.
    - `hooks` 배열: 상태 저장
    - `mount()`: 최초 렌더링
    - `update()`: 상태 변경 후 재렌더링

#### 제약조건

- `Hook`은 최상위 컴포넌트에서만 사용할 수 있습니다.
- `State`는 최상위 컴포넌트(루트 컴포넌트)에서만 관리합니다.
- 자식 컴포넌트는 상태를 가지지 않고, 부모에게 전달받은 `props`만 사용합니다.
    - 즉, 자식 컴포넌트는 `Stateless`한 순수 함수로 구현합니다.
    - `Lifting State Up` 패턴을 따릅니다.

### 2. State

- `State`는 변경 가능한 동적 데이터입니다.
- `State`가 바뀌면 화면이 자동으로 다시 렌더링되어야 합니다.

### 3. Hooks

- 함수형 컴포넌트에서도 상태와 생명주기 개념을 사용할 수 있게 하는 기능입니다.

#### 최소 구현 대상

- `useState`
- `useEffect`
- `useMemo`

#### 핵심 질문

> 함수는 매번 새로 실행되는데, 상태는 어떻게 유지될까?  
> 이 문제를 해결하는 것이 `Hooks`의 핵심입니다.

### 4. Virtual DOM + Diff + Patch

- Week 3에서 만든 `Virtual DOM`을 활용합니다.

#### 동작 흐름

- `Virtual DOM` 생성
- 이전 `Virtual DOM`과 비교(`Diff`)
- 변경된 부분만 실제 `DOM`에 반영(`Patch`)

> 목표: 전체를 다시 그리지 않고 필요한 부분만 업데이트하기

### 5. 테스트 페이지 개발

- Week 4와 Week 5에서 구현한 내용을 활용해, 브라우저에서 동작하는 웹 페이지를 구현합니다.
- 주제는 자유입니다.
- 단, 반드시 사용자 입력이나 클릭 등에 따라 화면이 변경되어야 합니다.

### 6. 기술 제한

- `JavaScript` 또는 `TypeScript`
- `HTML`, `CSS`
- 외부 프레임워크(`React`, `Vue` 등) 사용 금지

### 7. 구현 우선 원칙

- 이번 과제는 **학습보다 구현이 우선**입니다.
- AI를 적극적으로 활용해 **결과물을 만들어내는 것**이 최우선 목표입니다.
- 다만, 핵심 로직은 반드시 이해하고 설명할 수 있어야 합니다.
- 직접 작성이 어렵다면 AI로 전체 코드를 생성해도 괜찮지만, 생성된 코드를 반드시 학습하고 원리를 이해해야 합니다.
